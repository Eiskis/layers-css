Fluid grid

You need to mark the last column of the row, and using .clear after rows is a good idea, too. Columns are stacked from left to right, no wrappers are required. Columns within columns work fine.

Use n/12 or fractional keywords as class names, like this:

.column.six

.column.six.last

.column.four

.column.third

.column.four.last

.column.three

.column.three

.column.fourth

.column.fourth.last

.two

.one

.twofourths

.one

.two.last

You can .push- columns as well:

.column.two.push-one

.column.two.push-half.last

The stack can also start from the .right:

.column.right.five

.column.right.three

.column.right.two

Fixed-fluid column combination

One part gets a fixed width, the other side fills the rest. Very handy for combining images with a fluid grid. Multiple combinations can be nested. The fluid column always comes first in markup.

You can adjust the size of the fixed-width in steps with media queries, or break the combination completely.

.column.fluid > .column-content

.column.fixed

You can also stack this starting from the .right:

.column.fluid.right > .column-content

.column.fixed.right

Extend

This is the CSS you to adjust the size of the fixed column (default is 10em):


/*Fixed column width*/
.column.fixed {
	width: 10em;
}
.column.fluid > .column-content {
	margin-right: 10em;
}
.column.fluid.right > .column-content {
	margin-left: 10em;
}

Rows

.row-content is aligned to the center of the parent. .buffer also gives its content breathing room. You can easily set up a full-bleed site with alternating background styling for each section with this HTMl:

.row-content.buffer > .column.half

.row-content.buffer > .column.half.last

Controlling max-width of .row-content works nicely for full-bleed layouts:

.row-content.buffer > .column.full

You can naturally use any element (like section for example) for the rows. You can use rows to build both the overall frame of your site and individual page content. .row-content has a default maximum-width of 70em.

Common tools

Generic
.clear
Clear stuff (clear: both;)
.hidden
Hide stuff (display: none;)
.dry
Move inline content away from view
.border-box, .content-box
Switch box-sizing mode in all browsers.
.block, .inline-block, .inline
Change display property.
.float-left, .float-right
Float an element, remove clearing.
Margins & spacing
.reset
Reset margins to 0
.squeeze
Use smaller margins (half of the default)
.margin
Add default margins

Directions work as modifiers:

  • .reset-top, .reset-bottom
  • .squeeze-right
  • .margin-top
.buffer
Add comfortable paddings (like in this box)
.no-buffer
Remove any paddings

Directions work as modifiers:

  • .buffer-left.buffer-right
  • .no-buffer-top

Psst! You can automatically use these as mixins in LESS.

Forms

Native input element styles are left alone. Input fields use the same, default box-sizing model as all other elements in all browsers so they're easy to customize.

.plain elements are robbed of their OS-specific default styles (background-color added for clarity).

Remember, .buffer gives elements breathing room.
