Layers CSS reference

Here's a clean CSS package that makes it easy to pick the stuff that you need, without having to override things. It doesn't choose a visual style or color scheme for you, it just gives you tools. Native input elements preserved, no dependencies or requirements. (Non-semantic class names: they work, deal with it.)

Please note that the color scheme is on this page is included in extras, the main release doesn't include any colors. Some background colors are added for clarity. Take a look at the source code to learn more.

Look here to see how native elements look with nothing but Layers CSS loaded.

Fluid grid

You need to mark the last column of the row, and using .clear after rows is a good idea, too. No wrappers are required. Columns within columns work fine.

.column.eight

.column.four last

You can .push- columns.

.column.four

.column.three.push-two

.column.two.last

If you want to start the stack from the .right:

.column.right.five

.column.right.two.push-two

.column.right.three

Without gutters

.column.six.reset

.column.three.reset

.column.three.reset

You can also use proportional keywords:

.column.half.first

.column.fourth

.column.fourth

Fixed-fluid column combination

One part gets a fixed width, the other side fills the rest. Very handy for combining images with a fluid grid. Multiple combinations can be nested. The fluid column always comes first in markup.

You can adjust the size of the fixed-width in steps with media queries, or break the combination completely.

.column.fluid > .column-content

.column.fixed

You can also stack this starting from the .right.

.column.fluid.right > .column-content

.column.fixed.right

Generic
.clear
Clear stuff (clear: both;)
.hidden
Hide stuff (display: none;)
.dry
Move inline content away from view
.border-box
Switch to this box-sizing mode in all browsers.
.float-left, .float-right
Float an element, remove clearing.

Common tools

Margins & spacing
.reset
Reset margins to 0
.squeeze
Use smaller margins (half of the default)
.margin
Add default margins

Directions work as modifiers:

  • .reset-top, .reset-bottom
  • .squeeze-right
  • .margin-top
.buffer
Add comfortable paddings (like in this box)
.no-buffer
Remove any paddings

Directions work as modifiers:

  • .buffer-left.buffer-right
  • .no-buffer-top

Psst! You can automatically use these as mixins in LESS.

Extend

This is the CSS you to adjust the size of the fixed column.

/*Fixed column width*/
.column.fluid > .column-content {
	margin-right: 10em;
}
.column.fluid.right > .column-content {
	margin-left: 10em;
}
.column.fixed {
	width: 10em;
}

Forms

Native input element styles are left alone. Input fields use the same, default box-sizing model as all other elements in all browsers so they're easy to customize.

.plain elements are robbed of their OS-specific default styles (background-color added for clarity).

Remember, .buffer gives elements breathing room.
